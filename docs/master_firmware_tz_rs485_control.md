# ТЗ для мастера (STM32): управление слейвами и доставка уставок

## 1. Роль мастера
- Централизованное управление блоками через RS485/Modbus RTU
- Сбор телеметрии со слейвов
- Передача REMOTE-уставок и расписаний света в слейвы
- Передача данных и статусов на операторский ПК

## 2. Частота Modbus-опроса
- Опрос каждого слейва: 1 раз в 5 сек
- За один цикл мастер читает телеметрию блока целиком (+0..+8)
- Мастер не обязан опрашивать позиционеры каждые 200 мс
  (быстрый контур реализуется локально на слейве)

## 3. Heartbeat для автонома слейвов
- Отдельный heartbeat-кадр не обязателен
- Любой валидный Modbus-запрос мастера считается heartbeat для слейва
- Мастер должен обеспечивать регулярные валидные запросы, чтобы слейв не входил в AUTONOMOUS при штатной работе

## 4. REMOTE-уставки, которые мастер отправляет в слейв
- Целевые позиции:
  - WINDOWS_POS_A_TARGET
  - WINDOWS_POS_B_TARGET
  - CURTAIN_POS_TARGET
- Температурные уставки воды:
  - SP_WATER_RAIL
  - SP_WATER_GROW
  - SP_WATER_UPPER
  - SP_WATER_UNDERTRAY
- Расписания света REMOTE:
  - до 4 расписаний
  - каждое: ENABLE + ON_HHMM + OFF_HHMM

Важно:
- Автономные уставки/расписание слейва мастер не меняет

## 5. Процедура записи уставок в слейв
1. Сформировать пакет control-регистров (версия + payload + CRC)
2. Записать 0x10 в control-область слейва
3. Отправить APPLY_CMD
4. Прочитать APPLY_STATUS и ACTIVE_CTRL_VERSION
5. При ошибке -> повтор/событие/диагностика

## 6. Логика расписаний света (для валидации на мастере)
- В REMOTE допустимо до 4 расписаний
- Может быть активно несколько сразу
- Семантика: OR по активным интервалам
- Интервалы через полночь допустимы (start > end, пример 20:00-01:00)
- Расписания одинаковы для всех дней недели

## 7. Поведение при восстановлении связи
- Слейв после AUTONOMOUS возвращается в REMOTE после 3 успешных циклов связи
- Мастер после восстановления должен повторно синхронизировать REMOTE-уставки

## 8. Данные для ПК
- Мастер передает на ПК:
  - snapshot телеметрии
  - качество датчиков
  - layout блоков
  - статусы применения уставок
  - события/ошибки обмена со слейвами

## 9. Приемка мастера
1. Стабильный опрос слейвов каждые 5 сек
2. Корректная запись REMOTE-уставок в слейвы и подтверждение APPLY
3. Корректная работа с 4 расписаниями света в REMOTE
4. Корректная обработка интервалов через полночь
5. Корректная resync-подача уставок после восстановления связи
