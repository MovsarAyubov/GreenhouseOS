# Master-Slave Autonomy v2 (согласованный профиль)

## 1. Цель
Обеспечить устойчивую работу блоков при отказе мастера и снизить нагрузку на датчики.

## 2. Heartbeat по RS485
- Отдельный heartbeat-кадр не используется.
- Любой валидный Modbus-запрос мастера к слейву считается heartbeat.
- Таймаут heartbeat: `30 секунд` без валидных запросов.

## 3. Переходы состояний слейва
- Состояния:
  - `REMOTE` — управление по уставкам мастера.
  - `AUTONOMOUS` — локальное управление по зашитой программе.

Переходы:
1. `REMOTE -> AUTONOMOUS`:
   - условие: нет валидных запросов мастера `30 сек`.
2. `AUTONOMOUS -> REMOTE`:
   - условие: `3` подряд успешных цикла связи с мастером.
   - после возврата управление только по уставкам мастера.

## 4. Локальные уставки автонома
- Локальные уставки автономного режима:
  - записаны в прошивку слейва,
  - мастером не изменяются.
- При входе в автоном:
  - форточки закрываются полностью.
- Дальше локальная программа:
  - отопление по суточным уставкам (пример: 22C день / 18C ночь),
  - освещение по расписанию RTC (пример: 05:00..20:00),
  - управление шторами/форточками по локальной логике.

## 5. Частоты опроса датчиков на слейве
- `AIR_TEMP`, `AIR_HUM`: `5 сек`.
- `WATER_RAIL`, `WATER_GROW`, `WATER_UNDERTRAY`, `WATER_UPPER_HEAT`: `5 сек`.
- `WINDOW_POS`, `CURTAIN_POS`: `200 мс`.

## 6. Позиционное управление
- Мастер задает целевую позицию (например, 70%).
- Слейв локально управляет мотором до достижения фактической позиции по позициономеру.
- В системе есть механические концевые выключатели.
- При потере связи дополнительные аварийные действия не требуются.

## 7. Ограничения наблюдаемости на ПК
- Прямой связи ПК<->слейв нет.
- При отказе мастера онлайн-диагностика слейвов на ПК недоступна.
- После восстановления мастера — resync через мастер.
